# Human-in-the-Loop 診断レポートテンプレート

構成は「**① 全メトリクスの表 → ② 4 分割の分析**」。

表のルール:
- 列は `メトリクス | 説明（何を測るか） | 値 | 算出 | 根拠・備考`。**長い根拠は値セルに詰めず「根拠・備考」列へ**（読みやすさ）。
- 単位は揃える。割合系は **%**（`0.69` ではなく `69%`）。
- 数えられるものは数値、判断が要るものだけ 高/中/低。**「分散度」「Override」など数値化できるものは数値で出す**。
- 外部データ（人間の工数・損失額・後工程の結果）が要るものは載せない。LLM 推測の値は `算出` 列に「LLM 推測」と明記。

```markdown
# 🪢 Human-in-the-Loop 診断レポート

| 項目 | 値 |
| --- | --- |
| 集計期間 | <YYYY-MM-DD 〜 YYYY-MM-DD> |
| 走査セッション数 | <N> |
| 総 tool_use 件数 | <N> |

## ① メトリクス一覧

| メトリクス | 説明（何を測るか） | 値 | 算出 | 根拠・備考 |
| --- | --- | --- | --- | --- |
| 介在回数 | 人間の介在を要した回数（自発確認 + ツール実行停止） | <N> | 実測 | 内訳: 自発確認 <N> / ツール停止 <N> |
| ターン中断（参考） | 人間が生成を途中で止めた回数 | <N> | 実測 | casual な割り込み含む。必ずしも是正ではない |
| 必要性率 | 介在のうち本当に必要だった割合。低い=ラバースタンプ | 約 <X>% | LLM 推測 | <根拠> |
| 技術的回避可能率 | 設計・モデル改善で消せた介在の割合 | 約 <X>% | LLM 推測 | <根拠> |
| 待機時間 | エージェントが回答待ちでアイドルした時間 | 中央値 <X>s / 最大 <X> | 実測 | AskUserQuestion 発行→回答 |
| 割り込み分散度 | 介入が集中作業をどれだけ分断したか | <N> セッションに分散 / 平均 <X> 件 | 実測 | 散らばるほどコンテキストスイッチ大 |
| コンテキスト十分性 | 提示情報だけで人間が判断できたか | 高/中/低 | LLM 推測 | <根拠> |
| 判断覆し率（Override） | 人間が提案を修正・却下した割合 | 約 <X>% | LLM 推測 | <根拠: post_tool <N> 件中 ~<M> 件が是正> |
| リスク加重価値 | 介入対象の影響度で重み付けした重要度 | 高/中/低 | LLM 推測 | <根拠> |
| 自律稼働率 | 介入なく完走したセッションの割合 | <X>% | 実測 | <N> 中 <M> |
| 信頼キャリブレーション | 自信がない所で聞けているかの精度 | 高/中/低 | LLM 推測 | <根拠: 自発確認 vs ツール停止の比> |
| カスケード率 | 1 つの確認が追加確認を誘発した割合 | 高/中/低 | LLM 推測 | <根拠> |
| 粒度ミスマッチ率 | 「もっと早く聞いて」等と言われた割合 | 高/中/低 | LLM 推測 | <根拠> |

## ② 分析（4 分割）

観測した介在と自律実行を次の 4 区分に分類する。各見出しに具体例（`agent_asks` / `human_interrupts` の tool_use 分 / `post_tool_human_messages`）と LLM の判断を書く。

### 1. 適切に人間が介在した場面
### 2. 過剰な介在（確認過多）
### 3. 介在すべきだったのに素通りで実行された場面（リスク）
### 4. 適切に自律できた場面

## ③ 改善提案（診断 × 現状設定）

`~/.claude/settings.json` の現状と突き合わせる。**既に設定済みは「設定済み」と明記して再提案しない**。

### permissions.allow 追加候補（確認過多を減らす）

| 対象 | 根拠（診断のどこから） | 現状 | 採用判断 |
| --- | --- | --- | --- |
| `Bash(grep*)` | ②/④ で頻出・read-only・無介入 | allow 未登録 | read-only。追加で確認削減 |

### permissions.deny / ask 追加候補（危険を止める）

| 対象 | 種別 | 根拠 | 現状 | 採用判断 |
| --- | --- | --- | --- | --- |
| `Bash(git push --force*)` | deny | ③ | **既に deny 済み** | 追加不要 |
| `Bash(git add -A*)` | ask | ③ ツール停止 | 未登録 | 広域 add だけ ask |

### Stop hook 追加候補
deny で拾いにくい操作の事後検知。本期間で必要なものがあれば。

### CLAUDE.md / 初期指示 追加候補

| 是正の傾向 | 追記案 |
| --- | --- |

```

③ の最強の証拠は **ツール実行停止（`human_interrupts.tool_use_count` と `by_interrupted_signature`）**。ターン中断は casual を含むので ③ の根拠にしすぎない。改善提案は**現状設定との差分**で出し、既設定は再提案しない。設定の自動編集はしない。chat には全文を貼らず、パス + 3〜5 行サマリだけ返す。
